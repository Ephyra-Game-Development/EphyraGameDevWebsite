Shader Universe Simulation

For this project I was tasked with using a noise algorithm to generate a chunk of terrain using the OpenGL graphical framework. The task allowed for a lot of creative freedom and thus I decided to take it a step further than the basic ideas presented.

I used C++ to generate a set of Perlin noise in the form of a cube map and used the generated texture cube to modify vertices into a cubic planet and converted the cube to a sphere using linear interpolation of sphere maths.

I then applied an exponential distance-based tessellation algorithm to the terrain and wrote an algorithm for temperatures of the sphere based on moisture and proximity to the equator and used this to apply colour as if it was a real planet.

I added some Rayleigh scattering in a shader to simulate an atmosphere and used random variables to initialize the procedural planet with a second celestial body orbiting it with a minimal saturation to simulate a moon.

Custom Animation & Game Engine

For my final year development project, I developed a custom animation and game engine from scratch using C++, this engine was designed to be an equivalent to unreal engine / unity but with the key difference being that it is open source and freely available for commercial uses to avoid issues that have cropped up with the changes to payment model.

The development of this engine was conducted following an agile development methodology using 2-week sprints with story pointed goals defined at biweekly meetings.
